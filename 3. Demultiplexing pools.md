After trimmomatic we do demultiplexing of the samples.
In RAD-seq samples are pooled together in one sequencing lane and each sample has a barcode.
process_radtags reads the barcode and separates reads into individual samples and removes restriction enzyme bases.

The output files of R1_paired.fastq.gz and R2_paired.fastq.gz will be the input. 
We also need to upload the barcodes file for the programm to run. The format of the barcodes is e.g barcodes_SP.txt

```
#PBS -N process_radtags_loop
#PBS -l select=1:ncpus=4:mem=25gb:scratch_local=25gb
#PBS -l walltime=11:00:00

module add stacks


POOL_NUMBERS=("011" "179" "220" "256" "347" "455" "509" "526" "604" "625")


for POOL_NUMBER in "${POOL_NUMBERS[@]}"
do
    process_radtags -i gzfastq \
        -1 "/storage/brno2/home/kat/RadSeqData/trimmomatic_4bp_cut/SP${POOL_NUMBER}_1_paired_4bpcut.fq.gz" \
        -2 "/storage/brno2/home/kat/RadSeqData/trimmomatic_adapters_clean/SP${POOL_NUMBER}_2.paired.fq.gz" \
        -b "/storage/brno2/home/kat/RAD_Paph_Lim/barcodes_Lim3/barcodes_SP${POOL_NUMBER}.txt" \
        -o "/storage/brno2/home/kat/demultiplex/" \
        --renz-1 ecoRI --renz-2 mseI --threads 4 -c -q -r
done
```
The outputs are: sample1.fq.gz. So each file now contains reads from only one sample (individual).
