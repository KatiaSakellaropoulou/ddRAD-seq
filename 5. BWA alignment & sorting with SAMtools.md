input file of the loop : samplelist.txt (Each line is one sample name)
input file for the script: sampleA.1.fq.gz and sampleA.2.fq.gz

BWA compares each sequencing read to the reference genome and determines:

where theread maps
alignment quality
mismatches
insertions/deletions

```
while IFS= read -r i; do

cat > "$i.bwamem2.sh" <<EOF

#!/bin/bash
#PBS -N $i.bwamem2
#PBS -l select=1:ncpus=1:mem=20gb:scratch_local=100gb
#PBS -l walltime=01:00:00
#PBS -o $i.stdout
#PBS -e $i.stderr

trap 'clean_scratch' TERM EXIT

cd "\$SCRATCHDIR" || exit 1

module load bwa-mem2
module load samtools

ref_genome=/storage/brno2/home/kat/genome/GCA_964341245.1_ilLimPopu1.hap1.1_genomic.fna
output_dir=/storage/brno2/home/kat/align/
read1=/storage/brno2/home/kat/map_seq/${i}.1.fq.gz
read2=/storage/brno2/home/kat/map_seq/${i}.2.fq.gz

bwa-mem2 mem -t 1 "\$ref_genome" "\$read1" "\$read2" | samtools view -b - > "${i}.bam"

samtools sort -@ 1 -o "${i}_sorted.bam" "${i}.bam"
samtools index "${i}_sorted.bam"


mv "${i}_sorted.bam" "\$output_dir/${i}_sorted.bam"
mv "${i}_sorted.bam.bai" "\$output_dir/${i}_sorted.bam.bai"

EOF

done < samplelist.txt
```

Output of this step: PBS shell scripts.
Example: sampleA.bwamem2.sh
Output: SAM format
| samtools view -b - > "${i}.bam"
converts it to bam (Binary Alignment Map format) which is a compressed binary version of SAM.

What BAM contains
For every read:
chromosome position
alignment score
CIGAR string
mapping quality
sequence
flags


samtools sort -@ 1 -o "${i}_sorted.bam" "${i}.bam"

What sorting means
Reads are reordered by genomic coordinates.
Example:
chr1 position 100
chr1 position 101
chr1 position 102
instead of original sequencing order.

samtools index "${i}_sorted.bam"
What indexing does:
Creates a fast lookup index that allows software to instantly access: chromosome regions and 
specific genomic positions without reading the whole BAM.

Final important outputs

For each sample:

sampleA_sorted.bam (Coordinate-sorted alignments)

sampleA_sorted.bam.bai (Fast-access index.)
